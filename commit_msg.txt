Fix 5 mobile UX issues reported during manual testing

Issue 1 - Camera opens at top of screen (scroll jump)
  CameraModal now locks document.body overflow:hidden on mount and
  restores it on unmount. Prevents the underlying page from jumping
  to the top when the modal opens or closes.

Issue 2 - Capture button hidden below screen edge on iOS
  Portrait bottom bar and crop action bar now use
  padding-bottom: max(20px, env(safe-area-inset-bottom))
  so the buttons clear the home-indicator bar on iPhone X+.

Issue 3 - Silent failure after Crop and Scan
  backToCamera() now accepts a type parameter (info/warning/error).
  When validateScan returns valid:false, the feedback is shown as
  type error (red text, 6-second lock) instead of type info.
  Previously the rejection message was indistinguishable from the
  normal idle hint and users did not notice it.

Issue 4 - Wrong missing-fields count
  Removed the hardcoded count from the static localError string.
  The Status cell in the summary card now shows a live red count
  that updates as the user fills in fields, so the number is always
  accurate regardless of when the user reads it.

Issue 5 - Submission fails with generic error
  Added explicit 401 handling in SubmitSection catch block.
  When the session has expired (e.g. server restarted in dev, or
  4-hour TTL elapsed), the user now sees a clear message:
  "Your session has expired. Please refresh the page and start
  again from Step 1." instead of the generic failure message.

Also: remove "AI will extract..." subtitle from Document Scanning header.
